Análise de Risco de Crédito utilizando Machine Learning em Python¶

Em operaes que envolvem um tempo entre a posse do produto ou servio e o seu pagamento, normalmente, ocorrem em condies de crdito, portanto deve existir uma relao de confiana entre as partes envolvidas.

No entanto, tambm surge uma probabilidade do pagamento no ocorrer conforme o esperado, ou seja, o risco de crdito. Com isso, se faz necessrio a avaliao desse risco de crdito para evitar a inadimplncia, pois se muitos clientes comearem a no honrar com o pagamento, pode gerar grandes prejuzos para a instituio.

Assim, uma forma de avaliar o default (termo utilizado para indicar o no cumprimento das obrigaes e/ou condies de um emprstimo) utilizando modelos de Machine Learning.

Entendendo o problema¶

Neste projeto, vou utilizar um conjunto de dados para criar um modelo de Machine Learning para prever a probabilidade de um cliente do Nubank no pagar a fatura do seu Carto de Crdito.

Com isso, o objetivo desse projeto identificar a probabilidade de um cliente se tornar inadimplente. Ainda, vale lembrar que necessrio fazer essa avaliao no momento em que o cliente solicitar o carto de crdito.

Alm disso, com a construo do modelo espera-se que ele seja capaz de reduzir as perdas do Nubank com inadimplncia, mas tambm minimizar os falsos positivos.

O conjunto de dados est disponvel neste link.

Como podemos perceber, mesmo aps a remoo dos valores ausentes, a varivel target_default ainda apresenta-se bastante desbalanceada, posteriormente faremos o balanceamento.

Tambm h outras variveis com valores ausentes, no entanto como no temos muitas informaes sobre elas podemos assumir que:

  • Alguns campos no tm preenchimento no momento do cadastro, como, por exemplo, a varivel facebook_profile;
  • Algumas colunas podem indicar que os valores ausentes so, na verdade, valores nulos como so os casos das colunas last_amount_borrowed, last_borrowed_in_months e n_issues.

Sendo assim, para o segundo caso, vamos substituir os valores ausentes por zero. E o restante faremos das seguinte forma: para as variveis numricas vamos substituir os valores ausentes pelo valor da mediana e para as categricas utilizaremos o valor mais frequente.

In [20]:

col_cat = df_clean.select_dtypes(include='object').columns

df_clean['n_issues'].fillna(value=0, inplace=True)

# Substituindo valores ausentes nas variveis numricas
num_imputer = SimpleImputer(missing_values = np.nan, strategy='median')
num_imputer = num_imputer.fit(df_clean.loc[:, col_num])
df_clean.loc[:, col_num] = num_imputer.transform(df_clean.loc[:, col_num])

# Substituindo valores ausentes nas variveis categricas
cat_imputer = SimpleImputer(missing_values = np.nan, strategy='most_frequent')
cat_imputer = cat_imputer.fit(df_clean.loc[:, col_cat])
df_clean.loc[:, col_cat] = cat_imputer.transform(df_clean.loc[:, col_cat])

vis_rf = ROCAUC(lgbm_model, classes=['False', 'True'], title="Curvas ROC para o LightGBM")

vis_rf.fit(X_und, y_und); # Ajusta os dados de treino ao visualizador
vis_rf.score(X_test, y_test); # Avalia o modelo com os dados de teste
vis_rf.show(); # Finaliza e mostra a figura

  • A varivel objetivo target_default estava altamente desbalanceada, portanto foi realizado um balanceamento;
